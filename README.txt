ТЗ на сервис Secret
Описание сервиса
Сервис предназначен для безопасного обмена текстовыми данными. Например это могут быть доступы к сайтам и прочее.
Задание
Необходимо реализовать backend, frontend и api (api факультативно).
Сценарий использования
Пользователь А размещает секретный контент на сервисе. Защищает его паролем.
Передает пользователю Б ссылку на контент на сервисе. 
Передает пользователю Б пароль к контенту способом отличным от передачи ссылки. 
Пользователь Б заходит по ссылке. Вводит пароль от контента. Получает контент.
Примечания
Ссылка должна быть:
уникальная;
автогенерируемая;
имеет вид: домен/fgh8b5.
Информация должна сохраняться в БД и быть доступна для удаления пользователем, который знает пароль. Данные в БД должны быть зашифрованы.
